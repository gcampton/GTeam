# Community Launch Checklist

## Pre-Launch (before inviting anyone)
- [ ] Platform selected with clear rationale
- [ ] Server/workspace created with correct settings
- [ ] Channel structure set up (categories, names, descriptions)
- [ ] Role hierarchy configured with permissions tested
- [ ] Community rules written and posted in #rules
- [ ] Welcome message / DM configured and tested
- [ ] Moderation bot installed and configured (automod rules active)
- [ ] At least 2 moderators recruited and briefed
- [ ] 5-10 seed members invited for soft launch
- [ ] Seed content posted (3-5 discussion starters so it's not empty)

## Soft Launch (first 1-2 weeks, invite-only)
- [ ] Seed members active and providing feedback
- [ ] Channel structure validated (any channels unused? any missing?)
- [ ] Bot automation working (welcome DM sends, roles assign correctly)
- [ ] Moderation tested (can mods mute, warn, ban?)
- [ ] Rules clear and understood (any confusion from seed members?)
- [ ] Adjusted based on feedback

## Public Launch
- [ ] Invite link created (permanent or expiring?)
- [ ] Launch announcement posted on all owned channels (social, email, website)
- [ ] First 48 hours: monitor actively, respond to every new member
- [ ] Track: join rate, intro rate, first-message rate
- [ ] Post-launch retrospective after 1 week

# In-Product Onboarding Tactics

## Welcome Modal

- Single CTA pointing to the first-value action
- Include a "Skip" option — never trap users
- Keep copy under 50 words
- Show once on first login, never again after dismissal

## Tooltips

- Contextual: triggered by hover or first visit to a feature area
- Dismissible: always include an X or "Got it" button
- Progressive: reveal tooltips in sequence, not all at once
- Max 2 sentences per tooltip

## Progress Checklist

- 3-5 steps visible in a sidebar or dashboard widget
- Each step links directly to the action required
- Show completion state (checkmarks, progress bar)
- Celebrate completion: confetti animation, congratulatory message, or unlock notification
- Auto-dismiss after all steps complete, with option to revisit

## Empty States

- Never show a blank screen — always show sample data or a placeholder
- Include a clear CTA: "Create your first [item]" or "Import data"
- Show what the screen will look like once populated (screenshot or illustration)
- Link to help docs or a 60-second walkthrough video

## Tool Recommendations

| Tool | Category | Best For | Pricing Model |
|---|---|---|---|
| Appcues | No-code onboarding | Modals, tooltips, checklists without engineering | Per MAU |
| Userflow | No-code onboarding | Similar to Appcues, strong checklist builder | Per MAU |
| Intercom | Messaging + onboarding | In-app messages, product tours, help centre | Per seat + MAU |
| Hotjar | Session recording | Heatmaps, session replays, friction identification | Per session |
| FullStory | Session recording | Detailed session replay, frustration signals, search | Per session |

## Selection Criteria

- **Self-serve / SMB tier:** Appcues or Userflow for automated onboarding flows
- **Mid-market:** Intercom for combined messaging + onboarding
- **Enterprise:** Custom implementation; use session recording (Hotjar/FullStory) to validate
- **Budget-conscious:** Start with native empty states + progress checklists before buying tools

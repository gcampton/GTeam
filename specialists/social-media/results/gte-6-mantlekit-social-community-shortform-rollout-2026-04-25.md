## C) Community Response Playbook (Day-30)

## Daily engagement window

- Two blocks/day: 20 minutes each
- SLA targets:
  - Public comments: <4 hours
  - DMs: <12 hours
  - Technical setup questions: same day, with docs link

## Response framework

- Acknowledge -> clarify -> answer -> next step
- Keep answers specific and actionable
- If unknown, say "I will verify and follow up" and close loop within 24h

## Comment templates

- Praise:
  - "Appreciate that. If you want, I can share the exact setup checklist we use for launch-ready stacks."
- Feature question:
  - "Yes, MantleKit supports that. Short version: [1-sentence answer]. If useful, I can point you to the exact section."
- Objection (price):
  - "Fair question. The tradeoff is paying once vs rebuilding auth/payments/admin repeatedly."
- Comparison request:
  - "Good comparison. The biggest difference is MantleKit includes post-login operations, not just the frontend shell."

## Escalation rules

- Security or legal concern -> escalate to founder immediately
- Refund/account issue -> move to support channel with ticket reference
- Hostile thread -> one calm factual reply, then stop back-and-forth

## Reddit protocol

- 80% education, 20% brand mention
- No link drops unless directly relevant
- Use transparent language: "I built this" when sharing MantleKit
